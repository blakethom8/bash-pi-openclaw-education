# 1.4 Exit Codes and Error Handling

**How agents know if something worked**

---

## What Are Exit Codes?

Every command returns a number when it finishes:
- **0** = success
- **Non-zero** = error (1, 2, 127, etc.)

This is how agents (and scripts) know if something worked.

---

## Checking Exit Codes

```bash
# Run a command
ls /existing-folder
echo $?  # prints: 0 (success)

# Try something that fails
ls /nonexistent
echo $?  # prints: 2 (error)
```

**`$?`** = exit code of the last command

---

## Common Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | General error |
| 2 | Misuse of command |
| 126 | Command not executable |
| 127 | Command not found |
| 130 | Ctrl+C (terminated) |

---

## Conditional Execution

### `&&` (AND) — Run if previous succeeds

```bash
mkdir new-folder && cd new-folder
# cd only runs if mkdir succeeds
```

### `||` (OR) — Run if previous fails

```bash
cd /missing-folder || echo "Folder doesn't exist"
# echo only runs if cd fails
```

### Chain them

```bash
mkdir reports && cd reports && touch report.md || echo "Failed"
```

---

## Error Handling in Scripts

### Set error handling mode

```bash
#!/bin/bash
set -e  # Exit on any error
set -u  # Exit on undefined variable
set -o pipefail  # Exit if any command in pipeline fails

# Combine them
set -euo pipefail
```

**Why agents use this:** Scripts fail fast, don't continue with bad state.

---

### Manual error checking

```bash
#!/bin/bash

if ! command; then
  echo "Command failed!"
  exit 1
fi

# Or
command || { echo "Failed!"; exit 1; }
```

---

## Agent Pattern: Try/Fallback

```bash
# Try primary method, fall back if fails
curl https://api.example.com/data || curl https://backup-api.example.com/data
```

---

## Agent Pattern: Validate Before Proceeding

```bash
#!/bin/bash
set -euo pipefail

# Check prerequisites
if ! command -v jq &> /dev/null; then
  echo "Error: jq not installed"
  exit 1
fi

# Proceed with confidence
cat data.json | jq '.results'
```

## Practice

**Task:** Write a script that checks if a file exists, processes it if yes, errors if no.

<details>
<summary>Solution</summary>

```bash
#!/bin/bash
set -euo pipefail

FILE="$1"

if [ ! -f "$FILE" ]; then
  echo "Error: File $FILE not found"
  exit 1
fi

echo "Processing $FILE..."
cat "$FILE" | wc -l
echo "Done!"
```
</details>
