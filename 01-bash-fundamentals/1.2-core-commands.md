# 1.2 Core Commands (the ones agents actually use)

Agents don't need to know all 100+ bash commands. They need **~20 core commands** that cover 90% of tasks.

---

## File Navigation

### `ls` — List files

```bash
ls                  # list files in current directory
ls -la              # list all files (including hidden) with details
ls *.txt            # list only .txt files
ls -lh              # human-readable file sizes
```

**Agent use:** "What files are in this folder?"

---

### `cd` — Change directory

```bash
cd Documents        # go to Documents folder
cd ..               # go up one level
cd ~                # go to home directory
cd -                # go back to previous directory
```

**Agent use:** Navigate to the right folder before working.

---

### `pwd` — Print working directory

```bash
pwd                 # show current directory path
```

**Agent use:** "Where am I right now?"

---

## Reading Files

### `cat` — Read entire file

```bash
cat file.txt                # print entire file
cat file1.txt file2.txt     # concatenate multiple files
```

**Agent use:** Load file contents into context.

---

### `head` / `tail` — Read beginning/end

```bash
head -n 10 file.txt         # first 10 lines
tail -n 20 file.txt         # last 20 lines
tail -f logfile.txt         # follow file (watch for new lines)
```

**Agent use:** Preview large files, monitor logs.

---

### `less` — View file with scrolling

```bash
less large-file.txt         # scroll through file
                            # (press 'q' to quit)
```

**Agent use:** Inspect files without loading everything.

---

## Writing Files

### `echo` — Write text

```bash
echo "Hello World"          # print to screen
echo "Hello" > file.txt     # overwrite file
echo "World" >> file.txt    # append to file
```

**Agent use:** Create files, save output.

---

### `touch` — Create empty file

```bash
touch newfile.txt           # create empty file
touch file1 file2 file3     # create multiple files
```

**Agent use:** Initialize files before writing.

---

## File Operations

### `cp` — Copy

```bash
cp source.txt dest.txt      # copy file
cp -r folder1/ folder2/     # copy directory recursively
```

---

### `mv` — Move (or rename)

```bash
mv old.txt new.txt          # rename file
mv file.txt ~/Documents/    # move file to Documents
```

---

### `rm` — Remove

```bash
rm file.txt                 # delete file
rm -r folder/               # delete directory recursively
rm -f file.txt              # force delete (no confirmation)
```

⚠️ **Careful:** No trash/recycle bin. Deleted = gone.

---

### `mkdir` — Create directory

```bash
mkdir new-folder            # create directory
mkdir -p path/to/folder     # create nested directories
```

---

## Text Processing

### `grep` — Search text

```bash
grep "error" logfile.txt            # find lines containing "error"
grep -i "Error" logfile.txt         # case-insensitive search
grep -r "TODO" src/                 # search recursively in directory
grep -v "debug" logfile.txt         # invert match (lines WITHOUT "debug")
```

**Agent use:** Find relevant lines in large files.

---

### `sed` — Find and replace

```bash
sed 's/old/new/' file.txt           # replace first occurrence per line
sed 's/old/new/g' file.txt          # replace all occurrences
sed -i '' 's/old/new/g' file.txt    # edit file in-place (Mac)
sed -i 's/old/new/g' file.txt       # edit file in-place (Linux)
```

**Agent use:** Modify file contents programmatically.

---

### `awk` — Column/field processing

```bash
awk '{print $1}' file.txt           # print first column
awk -F',' '{print $2}' data.csv     # print second column (CSV)
awk '$3 > 100' data.txt             # filter rows where column 3 > 100
```

**Agent use:** Process structured data (CSV, logs, tables).

---

### `sort` — Sort lines

```bash
sort file.txt                       # sort alphabetically
sort -n numbers.txt                 # sort numerically
sort -r file.txt                    # reverse sort
sort -k2 data.txt                   # sort by 2nd column
```

---

### `uniq` — Remove duplicate lines

```bash
uniq file.txt                       # remove consecutive duplicates
sort file.txt | uniq                # remove all duplicates (must sort first)
uniq -c file.txt                    # count occurrences
```

---

### `wc` — Count lines/words/characters

```bash
wc -l file.txt                      # count lines
wc -w file.txt                      # count words
wc -c file.txt                      # count characters
```

**Agent use:** Quick stats on file size/content.

---

## Data Formats

### `jq` — JSON processor

```bash
cat data.json | jq '.'              # pretty-print JSON
cat data.json | jq '.users[]'       # extract array
cat data.json | jq '.name'          # get specific field
cat data.json | jq '.users[] | select(.active == true)'  # filter
```

**Agent use:** Parse API responses, process JSON data.

---

### `curl` — HTTP requests

```bash
curl https://api.example.com        # GET request
curl -X POST -d '{"key":"value"}' https://api.example.com  # POST
curl -o file.txt https://example.com/file.txt  # download file
curl -H "Authorization: Bearer TOKEN" https://api.example.com  # headers
```

**Agent use:** Call APIs, download files, web scraping.

---

## Scripting Languages

### `python` — Run Python code

```bash
python script.py                    # run Python script
python -c "print(2 + 2)"            # one-liner
echo "data" | python process.py     # pipe data to script
```

---

### `node` — Run JavaScript/TypeScript

```bash
node script.js                      # run Node script
node -e "console.log(2 + 2)"        # one-liner
```

**Agent use:** When bash isn't enough, drop into Python/Node for complex logic.

---

## System Commands

### `which` — Find command location

```bash
which python                        # shows: /usr/bin/python
which nonexistent                   # shows nothing (command not found)
```

**Agent use:** Check if a tool is installed.

---

### `chmod` — Change permissions

```bash
chmod +x script.sh                  # make executable
chmod 755 script.sh                 # rwx for owner, rx for others
```

**Agent use:** Make scripts runnable.

---

### `env` — Show environment variables

```bash
env                                 # list all variables
echo $PATH                          # show specific variable
export MY_VAR="value"               # set variable
```

## Real-World Agent Examples

### Example 1: Find errors in logs
```bash
cat /var/log/app.log | grep "ERROR" | tail -20
```
→ "Show me the last 20 errors"

---

### Example 2: Process JSON data
```bash
curl https://api.github.com/users/blakethom8 | jq '.name'
```
→ "Get Blake's name from GitHub"

---

### Example 3: Count files by type
```bash
ls *.txt | wc -l
```
→ "How many .txt files are here?"

---

### Example 4: Create report
```bash
echo "# Report" > report.md
echo "Date: $(date)" >> report.md
echo "Files processed: $(ls data/ | wc -l)" >> report.md
```
→ "Generate a markdown report"

## Key Patterns

**1. Chain commands with pipes:**
```bash
cat file | grep "text" | sort | uniq
```

**2. Save output to file:**
```bash
command > output.txt
```

**3. Conditional execution:**
```bash
command1 && command2  # run command2 only if command1 succeeds
```

**4. Loop over files:**
```bash
for file in *.txt; do
  cat "$file" | wc -l
done
```

## Practice Challenge

**Task:** Find all Python files in `src/`, count total lines of code.

**Solution:**
```bash
find src/ -name "*.py" -exec wc -l {} + | tail -1
```

Or step-by-step:
```bash
find src/ -name "*.py" > python-files.txt
cat python-files.txt | xargs wc -l
```
