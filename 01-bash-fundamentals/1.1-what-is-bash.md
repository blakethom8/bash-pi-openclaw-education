# 1.1 What is Bash? (and why agents love it)

## What is Bash?

**Bash** = **B**ourne **A**gain **SH**ell

Two things in one:
1. **Shell** — the program that runs your terminal commands
2. **Scripting language** — write programs using those commands

When you open Terminal on Mac/Linux, you're (usually) in bash.

---

## Why Do Agents Love Bash?

### 1. **Universal Interface**

Bash runs on every Unix system (Mac, Linux, servers):
- Same commands work everywhere
- No dependencies to install (usually)
- Direct access to the operating system

**For agents:** One tool to rule them all. No need to learn 50 different APIs.

---

### 2. **Everything is Accessible**

Through bash, you can:
- Read/write files (`cat`, `echo`)
- Run programs (`python`, `node`, `curl`)
- Process data (`jq`, `grep`, `awk`)
- Control the system (`mkdir`, `chmod`, `sudo`)

**For agents:** If it can be done on a computer, bash can do it.

---

### 3. **Composability**

Bash commands chain together with **pipes** (`|`):

```bash
# Chain together: read file → find lines → count them
cat logfile.txt | grep "ERROR" | wc -l
```

Small tools → complex workflows.

**For agents:** Build solutions from primitives, not monolithic apps.

---

### 4. **Self-Documenting**

Commands have built-in help:
```bash
man ls        # manual for 'ls'
ls --help     # quick help
tldr ls       # simplified examples (if installed)
```

**For agents:** Can look up how to use commands on the fly (just like you do).

---

### 5. **Extensible**

Can't do something with bash? Write a script:

```bash
#!/bin/bash
# my-custom-tool.sh
echo "Processing $1..."
# custom logic here
```

Now `./my-custom-tool.sh` is a bash command.

**For agents:** Create new tools as needed. Infinitely extensible.

---

## How Agents Use Bash

**Pi's 4 tools:**
1. **Read** — `cat file.txt`
2. **Write** — `echo "content" > file.txt`
3. **Edit** — `sed`, `awk`, or write script
4. **Bash** — run anything else

That's it. Everything else is just combinations of these.

---

## Bash vs. Other Shells

| Shell | Pros | Cons |
|-------|------|------|
| **bash** | Universal, stable, well-documented | Older syntax |
| **zsh** | Modern features, better completion | Less universal |
| **fish** | User-friendly, great defaults | Different syntax |
| **sh** | Minimal, guaranteed on all systems | Very basic |

**For agents:** Bash wins because it's **everywhere** and **predictable**.

---

## Example: What an Agent Sees

**You see:**
> "Summarize this podcast"

**Agent thinks:**
1. Download audio → `curl <url> -o podcast.mp3`
2. Transcribe → `whisper podcast.mp3 -o transcript.txt`
3. Summarize → `cat transcript.txt | llm "summarize this"`
4. Save → `echo "$summary" > summary.md`

**All bash commands.** No special APIs needed.

---

## Key Takeaway

> **Bash is the universal language of computers.**  
> If an agent can use bash, it can do anything a human programmer can do.

**That's why Pi says: "Bash is all you need."**
